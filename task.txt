Тестовое Backend
1. Контекст задачи
В рамках Бизнес-форума планируется интерактив для участников в виде геймификации посещения выступлений спикеров и стендов партнёров. Каждый участник форума получает доступ к Telegram Mini App, сканирует QR-коды на точках и копит баллы, которые затем можно потратить в магазине на призы.
 
Твоя задача – разработать RESTful бэкенд, который обслуживает эту логику: учет баллов, работу магазина и обмен баллов на товары.
 2. Функциональные требования
Ниже описаны обязательные модули и их требования.
 
№
Требование
Детали
1
Личный кабинет


Возврат текущего баланса баллов и полной истории начислений (дата, точка, количество баллов).
2
Начисление баллов


Пользователь сканирует QR-код в Mini App и отправляет токен на сервер. При успешной валидации токена баллы зачисляются однократно за каждую точку. Повторное сканирование той же точки — баллы не начисляются, возвращается понятное сообщение.
3
Магазин


Список товаров: название, описание, стоимость в баллах.
4
Обмен баллов на товар


Пользователь выбирает товар. Если баллов достаточно и товар в наличии — баллы списываются, количество товара уменьшается. Иначе — ошибка с понятным сообщением.
5
QR-коды для точек


Каждая точка имеет уникальный QR-код. Администратор может сгенерировать QR-код для точки и обновить его (ротация токена). QR-код содержит токен с ограниченным сроком действия (TTL задаётся в конфиге).
6
Авторизация


Пользователь авторизуется через Telegram Mini App (initData).
Backend должен проверить подпись Telegram и использовать telegram_id из авторизационных данных.
После авторизации все запросы должны выполняться с использованием JWT токена.



 
2.1. Подробнее: механика QR-кодов
QR-код – это основной способ регистрации посещения точки. Ниже описана ожидаемая логика работы:
•   	Генерация: по запросу администратора сервер генерирует уникальный токен, связывает его с точкой, сохраняет в БД и возвращает данные для отрисовки QR-кода (строку токена или готовый PNG).
•   	Обновление (ротация): Повторный запрос токена инвалидирует предыдущий токен и выдаёт новый. Это позволяет регулярно обновлять QR-коды на стендах (например, каждые 2 минуты), чтобы исключить передачу кода между участниками.
•   	TTL токена: Срок жизни токена задаётся через переменную окружения QR_TTL_SECONDS. По истечении TTL токен считается невалидным.
•   	Сканирование: Пользователь сканирует QR-код в Telegram Mini App и отправляет токен на сервер. Сервер проверяет существование токена, его срок действия и факт повторного использования.
•   	Ответы при ошибках: Невалидный токен → 400. Истёкший TTL → 400 с пояснением. Повторное сканирование → 409 «Вы уже посетили эту точку».
 
2.2. Бонусное задание (не обязательно, но будет плюсом)
